# AI Repository Summary

> This document provides a structured summary of the repository for AI agents and automated tooling.

## Repository Overview

| Property | Value |
|----------|-------|
| **Name** | workflow-orchestration-queue-charlie80-b |
| **Owner** | intel-agency |
| **Purpose** | Headless agentic orchestration platform that transforms GitHub Issues into autonomous "Execution Orders" |
| **Primary Language** | Python 3.12+ |
| **Package Manager** | uv |

## Key Components

### Services

| Service | Path | Description |
|---------|------|-------------|
| Notifier Service | `src/notifier_service.py` | FastAPI webhook receiver for GitHub events |
| Sentinel Orchestrator | `src/orchestrator_sentinel.py` | Background polling service that processes queued work items |

### Data Models

| Model | Path | Description |
|-------|------|-------------|
| WorkItem | `src/models/work_item.py` | Unified work item model with status and task type enums |
| TaskType | `src/models/work_item.py` | Enum: PLAN, IMPLEMENT, BUGFIX |
| WorkItemStatus | `src/models/work_item.py` | Enum: agent:queued, agent:in-progress, agent:success, etc. |

### Queue Implementation

| Component | Path | Description |
|-----------|------|-------------|
| ITaskQueue | `src/queue/github_queue.py` | Abstract interface for queue backends |
| GitHubQueue | `src/queue/github_queue.py` | GitHub Issues-based queue implementation |

## Testing Commands

```bash
# Install dependencies
uv sync --dev

# Run tests
uv run pytest

# Run tests with coverage
uv run pytest --cov=src --cov-report=term-missing

# Lint
uv run ruff check src tests

# Format check
uv run ruff format --check src tests

# Type check
uv run mypy src
```

## Project Conventions

1. **Code Style**: Ruff with line length 100
2. **Type Hints**: Required (mypy strict mode)
3. **Testing**: pytest with asyncio mode auto
4. **Imports**: Absolute imports from `src.`
5. **GitHub Actions**: All actions pinned to SHA

## Status Labels

| Label | Meaning |
|-------|---------|
| `agent:queued` | Work item waiting to be processed |
| `agent:in-progress` | Currently being executed |
| `agent:success` | Completed successfully |
| `agent:error` | Failed with recoverable error |
| `agent:infra-failure` | Infrastructure failure |
| `agent:reconciling` | Being reconciled |
| `agent:stalled-budget` | Budget limit reached |

## Security

- Webhook signature verification via HMAC SHA256
- Credential scrubbing before posting to GitHub
- Token patterns scrubbed: `ghp_*`, `ghs_*`, `gho_*`, `github_pat_*`, `Bearer`, `sk-*`, ZhipuAI keys
